# Contributing to MAANG System Design Playbook

Thank you for your interest in contributing! This playbook is designed to help engineers crack MAANG system design interviews.

## How to Contribute

### 1. Report Issues

- Found a mistake? Open an issue
- Have a suggestion? Open an issue
- Want to add a new case study? Open an issue first to discuss

### 2. Submit Pull Requests

**Before submitting:**

- Ensure your content follows the **11-step format** (see README.md)
- Keep the tone **interview-focused** and **concise**
- Include **specific numbers** (QPS, storage, latency)
- Explain **trade-offs explicitly**
- Avoid textbook explanations â€” focus on decision-making

**Format Requirements:**

Every case study MUST follow this structure:

1. Problem Statement
2. Clarifying Questions
3. Requirements (Functional + Non-Functional)
4. Scale Assumptions
5. High-Level Architecture
6. Core Components
7. Data Flow
8. Bottlenecks
9. Trade-offs
10. How to Explain This in an Interview
11. Common Mistakes

### 3. Style Guide

**Do:**

- Use numbered sections (1, 2, 3...)
- Include concrete examples with numbers
- Explain WHY decisions are made
- Acknowledge trade-offs
- Use bullet points for readability
- Keep sentences short and crisp

**Don't:**

- Write long essays
- Use generic explanations
- Skip trade-off discussions
- Forget to mention scale assumptions
- Use buzzwords without justification

### 4. New Case Studies

If you want to add a new case study:

1. Open an issue first to discuss
2. Follow the 11-step format exactly
3. Include realistic scale assumptions
4. Focus on MAANG-level complexity
5. Add to the README.md structure section

**Popular requests:**

- Design Instagram
- Design Uber
- Design YouTube
- Design Ticketmaster
- Design Distributed Cache
- Design API Rate Limiter (already exists)

### 5. Review Process

All PRs will be reviewed for:

- Adherence to 11-step format
- Technical accuracy
- Interview relevance
- Clarity and conciseness
- Trade-off explanations

## Questions?

Open an issue or start a discussion. We're here to help!

---

**Remember**: This playbook is for interview prep, not production system design. Keep it focused on what matters in a 45-minute interview.
